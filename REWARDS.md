# 기여자 표기와 리워드 시스템 설계

이 문서는 프로젝트에 기여한 선생님, 시민, 학생, 개발자, AI 보조자의 기여를 공정하게 기록하고 보상하기 위한 초안입니다.

## 기본 철학

- 교육 공공재를 함께 만든 사람을 보이게 기록합니다.
- 큰 기능뿐 아니라 검증, 오타 수정, 수업 적용 후기 같은 작은 기여도 인정합니다.
- 리워드는 금전 보상보다 **명예, 신뢰, 포트폴리오, 교육 공동체 기여 기록**을 우선합니다.
- AI가 도운 경우에도 최종 책임과 판단을 한 사람을 함께 기록합니다.

## 기여 유형

| 코드 | 유형 | 예시 | 기본 포인트 |
|---|---|---|---:|
| DATA | 데이터/API 추가 | 공식 문서, 테스트 URL, CSV 자동화, Pages 문서 추가 | 10 |
| VERIFY | 검증 | 링크/스키마/행 수/라이선스 확인, 오류 수정 | 5 |
| LESSON | 수업 자료 | 활동지, 난이도별 질문, 평가 루브릭 | 8 |
| CODE | 예제 코드 | Streamlit/분석 노트북 개선 | 8 |
| DOCS | 문서 개선 | README, 설명, 번역, 오타 수정 | 3 |
| FIELD | 현장 적용 후기 | 수업에서 써본 결과, 학생 반응, 개선 제안 | 7 |
| ISSUE | 좋은 제안 | 재현 가능한 이슈, 데이터 후보 제안 | 2 |

포인트는 자동 보상보다 “기여 규모를 가볍게 가시화하는 지표”입니다. 리뷰어가 조정할 수 있습니다.

## 기여자 표기 방식

### 1. CONTRIBUTORS.md

모든 병합된 기여자는 `CONTRIBUTORS.md`에 기록합니다.

기록 예시:

```md
| 이름/핸들 | 역할 | 주요 기여 | 링크 | 누적 포인트 |
|---|---|---|---|---:|
| @greatsong | 기획/리뷰 | 초중고 정보 교육 방향, 데이터 범위 제안 | GitHub | 20 |
```

### 2. 데이터셋별 기여자

각 데이터 페이지에 다음 필드를 둘 수 있습니다.

- 제안자
- 구현자
- 검증자
- 수업 적용자

### 3. 릴리스 노트

월 1회 또는 주요 업데이트마다 “이번 달 기여자”를 릴리스 노트에 기록합니다.

## 리워드 아이디어

### 즉시 가능한 리워드

- README/Pages에 기여자 명단 표기
- GitHub Release에 기여자 감사 문구 포함
- “검증 완료 데이터셋” 배지에 검증자 표기
- 수업 자료 페이지에 원 제안자 표기

### 커뮤니티형 리워드

- 월간 기여자 하이라이트
- “교실에서 바로 쓴 데이터셋” 인증 배지
- 우수 기여자를 공동 메인테이너로 초대
- 교육 워크숍/발표 자료에 기여자 이름 표기

### 후원/금전 리워드가 생길 경우

후원이 생기면 아래 우선순위를 권장합니다.

1. 데이터/API 유지비 또는 도메인/배포 비용
2. 수업 자료 제작자 소액 사례
3. 검증/문서화 기여자 리워드
4. 학생 프로젝트 공모/상품

금전 리워드는 투명성을 위해 별도 `FUNDING_LOG.md`에 수입/지출/기준을 기록합니다.

## 운영 프로세스

1. PR 또는 Issue에 기여 유형 라벨 부여
2. 리뷰어가 포인트 초안 산정
3. 병합 시 `CONTRIBUTORS.md` 업데이트
4. 월 1회 누적 기여 정리
5. Pages/README에 주요 기여자 반영

## AI 기여 표기 원칙

- AI는 도구로 표기하고, 사람 책임자를 함께 기록합니다.
- 예: “초안 작성: AI 보조 / 검토·수정: @teacher-id”
- AI가 생성한 데이터나 설명은 공식 문서/테스트로 검증된 경우에만 병합합니다.

## 남은 결정 사항

- 포인트를 누가 최종 승인할지
- 선생님 실명/별명 표기 선택 방식
- 학생 기여자의 개인정보 보호 기준
- 후원이 생길 경우 회계/집행 주체
